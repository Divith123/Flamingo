# Product Overview Template

Use this template structure for all 15 Flamingo product overviews.

## Required Sections:

### 1. Overview (200-300 words)
- Product purpose and positioning within the Flamingo ecosystem
- Target market and user personas
- Core value proposition (3-5 key benefits)
- Integration points with other Flamingo products

### 2. Market Analysis (800-1000 words)
- Market landscape and trends
- 5-7 competitor analyses using generic names (Competitor A, Leading Platform B, etc.)
- Competitive differentiation for Flamingo
- Feature comparison matrix (table format)

### 3. Core Features (2000+ words)
- 8-12 major feature areas
- Each feature with 3-5 sub-capabilities
- Detailed explanations of functionality
- Configuration options where applicable

### 4. User Stories (10-15 stories)
- Format: "As a [role], I want to [action], so that [outcome]"
- Acceptance criteria (5-8 bullet points per story)
- Cover all major user types and workflows

### 5. Wireframe Descriptions (4-6 key screens)
- Dashboard layout with component descriptions
- Key workflow screens
- Form layouts
- List/table views
- Detail pages

### 6. Integration Points
- Connections to other Flamingo products
- Data flow descriptions
- API/webhook capabilities
- Third-party integrations

### 7. Technical Considerations (optional)
- Architecture notes
- Security considerations
- Performance targets
- Compliance requirements

### 8. Success Metrics (optional)
- KPIs table with targets

### 9. Development Phases (optional)
- Roadmap for MVP through full feature set

## Writing Guidelines:

1. **Competitor Naming**: Use generic identifiers only:
   - Leading Competitor A, B, C...
   - Market Leader in [category]
   - Enterprise Platform X
   - Cloud Solution Y
   - Never use real company or product names

2. **Feature Descriptions**: Be specific and actionable
   - Use "The system allows users to..." or "Users can..."
   - Include configuration options
   - Describe error handling and edge cases

3. **User Stories**: Cover the full user journey
   - Admin/Setup stories
   - End-user stories
   - Manager/Approver stories
   - Reporting/Analytics stories

4. **Wireframes**: Describe visual layouts in detail
   - Screen structure (header, sidebar, main content)
   - Component placement
   - Interaction patterns
   - Mobile considerations

5. **Integrations**: Emphasize the interconnected ecosystem
   - Every product connects to at least 3 other Flamingo products
   - Describe bidirectional data flows
   - Explain real-time vs. batch synchronization
